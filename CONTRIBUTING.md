# 贡献指南

感谢您对项目的兴趣！我们欢迎各种形式的贡献，包括但不限于：

- 报告错误
- 提出新功能建议
- 提交代码改进
- 改进文档

在开始之前，请确保您已阅读并同意我们的[行为准则](CODE_OF_CONDUCT.md)。

## 如何贡献

### 报告错误

如果您发现了错误，请：

1. 检查[现有问题](../../issues)以确保错误尚未被报告。
2. 如果错误未被报告，请使用[错误报告模板](ISSUE_TEMPLATE/bug_report.md)创建新问题。

### 提出新功能建议

如果您有新功能建议，请：

1. 检查[现有问题](../../issues)以确保类似功能尚未被建议。
2. 如果功能未被建议，请使用[功能请求模板](ISSUE_TEMPLATE/feature_request.md)创建新问题。

### 提交代码改进

如果您想提交代码改进，请：

1. Fork 此仓库。
2. 为您的功能创建一个新分支 (`git checkout -b feature/amazing-feature`)。
3. 提交您的更改 (`git commit -m '添加一些惊人的功能'`)。
4. 推送到分支 (`git push origin feature/amazing-feature`)。
5. 创建一个[拉取请求](../../pull/new)。

在提交拉取请求之前，请确保：

- 您的代码遵循项目的代码风格。
- 您已经添加了必要的测试。
- 所有测试都通过。
- 您已经更新了相关文档。

## 代码风格

请遵循以下代码风格指南：

- 使用一致的缩进（建议2个空格）
- 使用描述性的变量和函数名
- 添加适当的注释
- 保持函数简洁和专注

## 提交信息

请使用清晰和描述性的提交信息：

- 使用现在时态（"添加功能"而不是"添加了功能"）
- 第一行应该简洁，描述主要更改
- 如果需要，添加更详细的描述

## 许可证

通过贡献您的代码，您同意您的贡献将在与项目相同的许可证下授权。
